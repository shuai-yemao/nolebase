## ESP 8266 连接云端

1. 通过 USB 转 TTL 将 ESP 和电脑串口助手通信
	1. 发送 AT 指令（要勾选发送新行，也就是回车换行符），测试模块是否可以正常工作
	2. 打开 EMQX 部署服务器 [部署教程](https://docs.emqx.com/zh/cloud/latest/connect_to_deployments/mqttx.html)
		1. 安装 Docker [Docker快速安装]([about:blank](https://www.bilibili.com/video/BV1wWR2YjEb8/?spm_id_from=333.337.search-card.all.click&vd_source=6f77320ec3e6e86d4e2e004a411d3f96))
		2. Docker 部署 EMQX [Docker安装emqx文档](https://docs.emqx.com/zh/emqx/latest/deploy/install-docker.html)
		3. 使用 Dashboard 网页管理 [EMQX Dashboard网页](http://localhost:18083/)
		4. 因无法找到本地部署 MQTT 的 broker 已停用
	3. ESP 8266 连接阿里云平台 [ESP8266连接阿里云教程](https://blog.csdn.net/m0_56399733/article/details/139878469)
		1. AT+CWMODE=1//设置为 STA 模式
		2. AT+RST//重启生效
		3. AT+CWLAP//查看周围 AP（WIFI 热点）
		4. AT+CWJAP="wifiname","password"//加入 WiFi 热点//==注意不要使用 WiFi6 和 5G 频段，也不要有空格==
		5. 配置 MQTT 用户属性：AT+MQTTUSERCFG=0,1,"clientId","username","password",0,0,""（==AT+MQTTUSERCFG error 需要重新刷固件==，在 clientID 中的逗号前都需要加\，才能发送成功）
			1. 阿里云平台在设备界面可以查看 ![[Pasted image 20260412134323.png]]
		6. 连接指定 MQTT broker： AT+MQTTCONN=0,"mqttHostUrl", port,0
		7. 查询 MQTT 连接状态：     AT+MQTTCONN?
		8. 订阅设备数据：AT+MQTTSUB=0,"/sys/ProductKey/DeviceName/thing/service/property/set",1；AT+MQTTSUB=0,"/sys/ProductKey/DeviceName/thing/event/property/post_reply",1（==ProductKey，DeviceName 都需要根据设备来更换==）![[Pasted image 20260412134650.png]]
		9. 向阿里云平台上传属性数据 AT+MQTTPUB=0,"/sys/ProductKey/sDeviceName/thing/event/property/post","{\"method\":\"thing.event.property.post\"\,\"params\":{\"TEMP\": 58.2\,\"PH\": 19.4\,\"EC\": 49.8\,\"FAN\": 1\,\"ADD_WP\": 1}}",0,0
	4. Esp 8266 连接 STM 32 测试串口通信是否正常
		1. 串口接收数据正常
		2. 但是未检测到 OK 返回，因为 DMA 内存自增，strstr 未在 buf 数组来得及查找的 OK
		3. `strstr` 把 RX_BUF 当成 C 字符串处理，**遇到第一个 `\0` 就停止搜索**。
		4. 使用 memem 函数, 自己定义即可
		5. RST 后，串口中断被关闭？为什么？
		6. 缓冲区大小需要 256 ，不然发送指令不完全
		7. ==Clined_ID 和 password 需要同一时间生成的，不然 AT+MQTTCONN=0,"iot-06z009zxkguhoc6.mqtt.iothub.aliyuncs.com",1883,0 会失败==
		8. 上传数据 error 为什么？
			1. 格式错误如：\格式不对发送的 AT 指令中不含有\
			2. 发送字节过多，栈溢出，触发 hardfault 中断，在启动文件中调整栈大小即可
			3. SP8266 MQTT AT 指令的 payload 默认限制通常是 **256 字节**，分多次发或者减少一次发送字节
	5. Auto 标志位，置一上位机发送阈值，置零发送继电器标志位，完成后下位机发回上位机
	6. 日常发送传感器数据还要有设备编号
	7. 报错数据上传需要分清为何种错误和错误类别（高还是低）
	8. 上传测试完毕（代码参考 task. C 文件），测试接收处理
		1. 了解上位机向下发送数据包的格式
		2. 使用串口转 TTL 串口助手将 ESP 8266 连接云平台查看云平台发送的数据（实时调试界面点击设置按钮发送数据）格式 ![[Pasted image 20260414125122.png]]
		3. 根据其数据格式编写云平台接收处理函数，因为不断上传数据，导致缓冲区数据会被覆盖？尝试进入云平台接收处理函数时关闭 DMA 传输，观察缓冲区数据，不可行会导致 ESP 8266 上传数据失败
		4. 上传接收完毕 ESP 数据后，停止 DMA 传输调用云平台接收处理函数，其中 Strstr 查找字节过多以及 atoi 函数会进入 hard fault 中断（[hard faul中断问题定位方法](https://zhuanlan.zhihu.com/p/1900846917212177900)）![[Pasted image 20260414143341.png]] 打印寄存器数据观察
		5. 将 cmd 数组转换为静态放入 data 或者 bss 中不占用栈区，printf 也需要
