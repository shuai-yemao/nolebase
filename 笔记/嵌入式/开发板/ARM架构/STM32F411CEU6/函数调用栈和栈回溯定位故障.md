## 2. 逐点精讲

### 知识点 1：栈在程序调用过程中的变化

#### 辅助图示

1. CPU 基本组成 ![[Pasted image 20260323130352.png]]
2. 指令执行流程 ![[Pasted image 20260323131122.png]]
3. 程序在内存中的分段 ![[Pasted image 20260323132911.png]]

#### 核心逻辑/原理

1. 压栈，将函数的局部变量（生命周期与栈帧绑定），调用者 main 的返回地址（第 3 行）以及栈基址压入栈中
2. 为被调用者 funcA 创建栈帧，sp 下移，分配局部变量空间
3. 将被调用者 funcA 返回值存入寄存器，销毁被调用者栈帧
4. Sp 上移到调用者 main 的返回地址，跳回 main 第三行继续运行
5. Main 返回，最后的栈帧销毁，os 回收进程资源，调用栈（PSP）清空

#### 关键公式/结论

1. ==PC 也叫程序计数器，保存着将要执行的指令的地址==，PC 位宽与主存地址总线位宽相同
2. AR 也叫地址寄存器，保存着 CPU 访问主存的单元地址，==无论 CPU 是取指令还是存取数据,都必须先将要访问地址送入 AR,直到读写操作完成==
3. ==IR 也叫指令寄存器，保存着当前正在执行的指令==，从主存中取出的指令字存放在 IR 中,其位宽和指令字相同
4. GR 也叫通用寄存器，有多种用途，增加通用寄存器的数量,既可减少访问主存的次数,从而提高 CPU 的处理效率,又可以方便汇编编程以及编译器生成代码
5. PSW/PSR 也叫程序状态字寄存器，用于保存由算术运算指令、逻辑运算指令、测试指令等建立的各种条件标志，另外==程序状态字寄存器还可用于保存中断和系统工作的状态信息==,以便 CPU 能及时了解计算机运行的状态,从而便于控制程序
6. ID 也叫指令译码器，与时序信号、条件及状态信息进  行组合,形成各种具有严格时间先后顺序的操作控制信号 (即微操作控制信号序列)
7. 计算机内存分为三部分 - 栈区、堆区、全局区
8. **栈区**，存放局部变量，函数参数，当前状态，函数调用信息等，向地址减小的方向增长，所以是==栈顶地址不变，新元素是从栈底不断进入的，所以栈底地址不断减小的==，非常非常重要，可读可写可执行，都由系统自动动态的在这个栈内分配和释放）
9. **堆区**，由程序员手动 malloc 函数分配然后 free 释放，向地址增大的方向增长，由链表实现的，可读可写可执行，
10. **全局区**别名静态存储区，==其中的变量常量在程序运行期间会一直存在，不会释放，程序结束后由系统释放==。又可以分为两个小区
	1. 区 1（data 段）：存储初始化的全局变量和静态变量，可读可写不可执行；
	2. 区 2（. Bss 段）：未初始化的全局变量和未初始化的静态变量；
	3. 区 3（常量区）：是否加了 const 修饰的，所有字符串常量也是存储在静态存储区（注：在单片机中，是否加了 const 修饰的字符串常量都是放在 rom（flash）里面的，可能是因为单片机做了这样的存储优化，使得可以腾出全局区的内存来）
11. ==SP（Stack Pointer，栈指针）——始终指向栈的 " 当前顶端在哪里== "。
12. 栈的底端由 RBP（或 32 位系统里叫 EBP）标记，也叫栈基址，进入函数后就固定不动
13. 栈帧是指每个函数在栈上占据一块属于自己的区域，里面存放：这个函数的局部变量（比如 int a = 10）；保存下来的寄存器值（比如旧的 RBP）；调用方留下的返回地址
14. Heap 和 Stack 相向生长，Heap 从低地址往高地址扩，Stack 从高地址往低地址缩，两者共享中间那块空闲内存。==如果程序递归太深或者 malloc 太多，两者会撞在一起，就是常说的栈溢出或内存耗尽。==
15. Code / Data / BSS 三段大小在编译时就确定了，运行期间不会变；Heap 和 Stack 则是运行时动态伸缩的。

### 知识点 2：栈回溯调试

#### 应用场景

在系统进入异常中断后无法利用 LR 地址定位到错误

#### 辅助图示

1. Keil 调试模式的 CPU 寄存器窗口 ![[Pasted image 20260323195028.png]]
2. Keil 调试模式的内存查看窗口 ![[Pasted image 20260323195213.png]]
3. SRAM 中双栈的分布 ![[Pasted image 20260323195557.png]]
4. AAPCS![[Pasted image 20260323203143.png]]

#### 核心逻辑/原理

利用 sp 栈指针定位到被强行保存在栈帧中的返回地址

#### 关键公式/结论

1. 当触发异常时，处理器会暂停当前任务，进入 handler 模式
2. ==根据 LR 的异常返回值来判断栈指针是指向进程栈还是主栈，也决定了压栈的位置==
3. Sp 指向最新压入的寄存器（R 0 的地址）
4. 低位在前是小端模式 LSB
5. 压栈的顺序是依次把 xPSR, PC, LR, R12 以及 R3‐R0 由硬件自动压入适当的堆栈中，顺序不可改变
6. PSR: IPSR 位段 (地处 PSR 的最低部分) 会被更新为新响应的异常编号
7. ==栈帧销毁后内存并不清零。`ret` 只是把 SP 移走，那块内存的数据还在，只是下次有新函数调用时会被直接覆盖==
8. 递归的本质就是自己调用自己，每次调用都会压一个新栈帧。递归太深压的帧太多，SP 一直下移直到撞上 Heap，就是**栈溢出**（Stack Overflow）。

## 4. Q&A

### Q 1：如何理解函数调用栈？

 A 1: 函数调用栈分为入栈（call 指令）和弹栈（ret 指令），入栈会将调用者的返回地址和函数的局部变量以及栈基址压入栈（一般为 MSP）中，在 main 函数内存地址的上方（栈顶指针由高到低），弹栈就是返回值写入寄存器，sp 上移，销毁栈帧，从栈中取回返回地址跳回

### Q 2：C 语言空间分为几段？

A 2: 栈区，堆区，. Data 段，. Bss 段，代码区

### Q 3：内存分配中堆区和栈区的区别？

A 3: 栈区是由系统自动分配和释放内存的区域，而堆区需要程序员自己分配和释放内存

### Q 4：内存分配中（. Data 段）和（. Bss 段）的区别？为什么要分这两个段？

A 4:.

1. Data 段存储了代码中已经初始化的全局变量和静态变量，. Bss 段存储了代码中未初始化的全局变量和静态变量
2. .bss 段因为会被系统全部初始化为零，而在文件中只需要记录. Bss 段需要分配多少内存这一句话即可，大大节省了内存空间

### Q 5：为什么传给函数参数时，尽量不要超过 4 个？

A 5: 因为函数的前四个形参会被传入寄存器中（架构调用规定 AAPCS），多余的函数形参会被压入栈中而不是寄存器，而寄存器在 CPU 内部，访问速度极快，几乎是零延迟；栈在内存里，访问速度慢几十倍甚至上百倍。
