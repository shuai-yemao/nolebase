## 2. 逐点精讲

### 知识点 1：GPIO 的基本结构

#### 实际意义

 **让软件能控制硬件、读取硬件状态**。接口是 CPU 和 I/O 设备之间交换信息的媒介和桥梁。

####  应用场景

1. **输出控制**：[[LED]]、蜂鸣器、继电器、小电机（[[PWM]] 控制 5 v 直流小电动机）
2. **输入检测**：按键、开关、传感器数字信号
3. **外部中断**：快速响应外部事件（按键、触发信号）
4. **通信模拟**：软件模拟 [[I2C]]、[[嵌入式/通信协议/有线通信协议/SPI|SPI]]、1-Wire（aht 20） 等
5. **[[PWM]] 输出**：调光、调速、舵机控制
6. **电平转换**：开漏 + 上拉实现不同电压域信号转换

#### 常见误区

1. **浮空输入很安全**： 浮空状态下引脚电平不稳定，易受干扰产生误触发，应配置上拉 / 下拉电阻
2. **开漏输出可直接输出高电平**：开漏输出只能拉低电平，输出高电平需外部上拉电阻
3. **上拉 / 下拉电阻可替代外部驱动**： 上拉 / 下拉电阻提供的电流很小（μA 级），仅用于稳定电平，不能驱动负载

#### 辅助图示

![[Pasted image 20260307212803.png]]

#### 通俗人话解释

1. 当配置为**输出模式**时，它是 " 手 "，可以主动输出高电平（如 3.3V）或低电平（如 0V），用来控制外部设备（像开关灯一样）
2. 当配置为**输入模式**时，它是 " 脚 "，可以感知外部信号（如按键是否按下），并将这些信息传递给芯片
3. 当配置为**复用模式**时，它变成了 " 专用工具 "，可以实现特定功能（如串口通信、[[嵌入式/通信协议/有线通信协议/SPI|SPI]] 通信，也可以实现正常 IO 功能）
4. 上拉 / 下拉电阻就像 " 稳定器 "，确保 GPIO 在没有外部信号时也有确定的状态（很重要，可以防止信号输入和输出较为标准，但是阻值大小也会影响信号）
5. 保护二极管则像 " 安全气囊 "，防止外部电压过高或过低损坏微控制器

#### 核心逻辑/原理

 用寄存器配置为 **输入 / 输出 / 复用 / 模拟**各种不同的模式

#### 关键公式/结论

1. GPIO 也叫并行 I/O，I/O 设备分为块设备和字符设备（区别在于块地址可以寻址）
2. CPU 与 I/O 设备控制器中的寄存器和数据缓冲区的通信方式是将所有寄存器映射到内存空间，称为内存映射 I/O。
3. 开漏输出适合实现多主机总线（I 2 C）、电平转换
4. 推挽输出可直接驱动小负载（[[LED]] 等），无需上拉电阻
5. 根据 GPIOx_PUPDR 寄存器中的值决定是否打开弱上拉电阻和下拉电阻

### 知识点 2：GPIO 的输入模式

#### 辅助图示

![[Pasted image 20260308191625.png]]

#### 通俗人话解释

我每过五分钟分钟（1 个 AHB 时钟周期 ）就看一下烧水壶（输入数据寄存器）中的温度（电平信号）

#### 核心逻辑/原理

1. 根据 GPIOx_PUPDR 寄存器中的值决定是否打开上拉和下拉电阻
2. 输入数据寄存器每隔 1 个 AHB1 时钟周期对 I/O 引脚上的数据进行一次采样

#### 关键公式/结论

1. 输入模式分为上拉，下拉，浮空，模拟四种
2. GPIO 初始状态都是输入模式，且为高阻抗模式
3. GPIO 输入模式下的高阻抗，也就是浮空状态
4. 浮空模式易受外界影响产生漏电流，导致 MCU 功耗增加
5. 上下拉电阻的作用就是稳定 IO 口的电平信号
6. 施密特触发器的作用是稳定输入电平信号

#### 辅助图示

1. 开漏模式下驱动 [[LED]] ![[Pasted image 20260308193932.png]]
2. I 2 C 开漏输出 ![[Pasted image 20260308200958.png]]
3. 推挽模式下驱动 LED![[Pasted image 20260308202311.png]]

#### 通俗人话解释

1. 开漏输出模式就是一个开了口子的水桶，可以自己放水，但是要别人加水
2. 推挽输出模式就是一个秋千，可以向前推也可以往后挽

#### 核心逻辑/原理

通过控制 MOS 管的关断以及上下拉电阻的设置来控制 IO 口输出电平

#### 关键公式/结论

1. I 2 C 总线上接的都是开漏输出，只要其中某个引脚漏极为低电平，则整条线路都为低电平，也被称为“线与”。通过这种特性，I 2 C 从设备来获取总线控制权
2. 开漏模式: 输出寄存器中的“0”可激活 N-MOS,而输出寄存器中的“1”会使端口保持高组态 (Hi-Z)(P-MOS 始终不激活)
3. 推挽模式: 输出寄存器中的“0”可激活 N-MOS,而输出寄存器中的“1”可激活 P-MOS。

## 4. Q&A

### Q 1: 什么是 Vth (阈值电压)？

 A 1: MOS 管由栅极控制开断，当 VGS（漏极和源极的电压差）<Vth 则 MOS 断开

 同理 VGS>Vth 则 MOS 打开

### Q 2：开漏输出为什么叫开漏输出？它有什么特点？

A 2: 开漏输出就是把 MOS 管的漏极作为输出引脚，N-MOS 打开则输出低电平，N-MOS 关断为高阻态模式（浮空）；其特点为**可以轻松实现不同电压域之间的信号传输（3.3 v MCU 控制 5 v 外设），无需额外电平转换芯片**

### Q 3：解释 I 2 C 总线相连的设备 IO 口为什么要使用开漏输出模式而不是推挽输出？

A 3:

1. 使用开漏输出可以使多个从机共享总线，因为开漏输出从机都只能拉低电平，不会使输出冲突（一个从机输出高电平，一个输出低电平，主机无法识别），其次上拉电阻通过电阻分压的原理来使不同的电压等级（3.3 v 和 5 v）的从机共享总线，电阻大小会影响 MCU 功耗和上拉速率，选择适合电压等级的上拉电阻（**浮空状态 IO 口输出高电平由上拉电压相连的电源电压决定**）也是很重要的
2. 使用推挽输出最大的问题就是输出冲突，因为推挽输出可以主动的输出高低电平，不同设备输出不同电压会形成大电流导致短路，其次还会影响信号完整性，推挽输出最高只可以输出 3.3 v 无法兼容其他电压等级的从机
