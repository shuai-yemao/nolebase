#### 通俗人话解释

异常是上课突然肚子疼导致我心情烦躁，中断是上课突然打来的电话导致我心情烦躁

#### 核心逻辑/原理

CPU 内部（NVIC）打断当前程序执行流的叫异常（也叫内部中断或者软件中断），CPU 外部（EXTI）促使 CPU 内部 (NVIC) 产生异常的叫中断，异常和 CPU 是同步的，中段和 CPU 是异步的

#### 关键公式/结论

1. 通过使用软件触发中断寄存器 STIR 可以产生软件中断
2. 中断优先级分为抢占优先级和响应优先级，由 AIRCR 控制有 4 位，可设置分组，一般设置为 4 位（0~15）抢占优先级，数字越小优先级越高
3. 中断的全过程
	1. 外设发来中断请求（EXTI 或者 NVIC 中断屏蔽寄存器相应位置一，使能中断）
	2. 中断请求位挂起（EXTI 挂起中断请求寄存器）
	3. ==发送到 NVIC 中先判断其中断优先级来决定是否悬起（中断不能立即得到响应）还是响应==
	4. 当悬起状态在处理器响应之前被软件清除，则中断取消
	5. 当中断得到响应之后，悬起位会被硬件自动清零，CPU 由线程模式进入 handler 模式
	6. ==如果中断持续请求，该中断结束后会被再次置为悬起状态进行下一次中断响应==
	7. 如果某个中断在得到响应之前==（悬起位还未清除）==,其==请求信号以若干的脉冲的方式呈现,则被视为只有一次中断请求（丢中断的一种情况）==
	8. ==中断请求释放了（悬起位已清除）==,但是在服务例程返回前又重新被置为有效, 则处理器会记住此动作,重新悬起该中断，==而在中断运行程序当中软件清除了悬起标志位就会导致丢中断的现象出现==
	9. 每个外部中断都有一个活动状态位。在处理器执行了其 ISR 的第一条指令后,它的活动位就被置 1,并且直到 ISR 返回时才硬件清零
4. 在 CM3 中,若存储器系统够快,且==总线系统允许入栈与取指同时进行,同时该中断可以立即响应,  则中断延迟是雷打不动的 12 周期 ==(进行了入栈、取向量、更新寄存器以及服务例程取指的一系列操作，T=中断屏蔽 + 中断嵌套 +SBus 压栈 +IBus 取向量)
5. ==处理咬尾中断时,省去了堆栈操作,因此切入新异常服务例程的耗时可以短至 6 周期。==
6. 中断响应序列（开始中断前的准备工作）
	1. 入栈，==依次把 xPSR, PC, LR, R12 以及 R3‐R0 由硬件自动压入适当的堆栈中==：如果当响应异常时,当前的代码正在使用 PSP,则压入 PSP,即使用线程堆栈; 否则压入 MSP,使用主堆栈（使用数据总线）
	2. 取向量，从向量表中找出正确的异常向量,然后在服务程序的入口处预取指==（与入栈是同时进行的，指令总线的任务）==
	3. 更新寄存器，在入栈和取向量的工作都完毕之后,执行服务例程之前,还要更新一系列的寄存器, ==在进入异常服务程序后,LR 的值被自动更新为特殊的 EXC_RETURN,这是一个高 28 位全为 1 的值==

#### 实际操作步骤

1. 外部中断回调函数设计
	1. 通过 EXTI 边沿触发来判断按键是否按下
	2. 按键按下发送下降沿触发中断
	3. 开始计时并通过队列传递时间和触发信号
	4. 转换中断为上升沿触发
	5. 按键松开发送上升沿触发中断
	6. 结束计时并通过队列传递时间和触发信号
	7. 转换中断位下降沿触发
2. 按键任务设计
	1. 创建按键参数结构体并初始化
	2. 判断触发信号，当触发信号为下降沿
	3. 复制按键按下开始时间
	4. 判断触发信号，当触发信号为上升沿
	5. 复制按键按下结束时间
	6. 判断按键按下时间间隔小于 20 ms 为短按，反之为长按
	7. 发送按键事件给按键事件队列
3. [[LED]] 任务设计
	1. 判断按键事件，短按 [[LED]] 翻转一次，长按 [[LED]] 闪烁 3 次
4. 实际问题
	1. ==发送给队列的消息和接收的消息对不上？==, 很可能是发送和接收两方数据类型不对 ![[Pasted image 20260319141203.png]]
	2. 为什么要使用 xHigherPriorityTaskWoken 参数？中断内不会自动切换到更高优先级的任务。相反，设置了一个变量来==通知应用程序编写者应该执行上下文切换==，xHigherPriorityTaskWoken 指向的变量在==第一次使用之前必须初始化为 pdFALSE==。
	3. 为什么中断中使用 printf 函数会导致系统卡死？因为 printf 中重定向加了开启和关闭任务调度的函数，==中断中只能调用 FromISR 函数==

## 4. Q&A

### Q 1：什么是软中断？软中断有什么作用？什么时候可以使用软中断？

A 1:

1. 使用软件触发中断寄存器 STIR 产生的中断就是软件中断
2. 软件中断可以用于任务之间的同步以及切换；用于处理系统异常；用于事件通知
3. 通常用于大数据处理；驱动逻辑如状态机；==较为耗时的事件==

### Q 2: 什么是丢中断？为什么会丢中断？哪些情况会丢中断？

A 2:

1. " 丢中断 " 是指**硬件产生了中断信号，但 MCU 没有执行对应的处理逻辑**，导致这次中断事件被忽略或漏处理。
2. 中断悬起位未清除时，对发来的中断处理信号，处理器只能识别出一次中断
3. 悬起位挂起，MCU 处于屏蔽；其次就是悬起位消除后再挂起但是被软件清除导致的丢中断

### Q 3：处理异常时，在入栈之后，LR 起了什么作用？LR 存了什么值？被打断程序的 LR 的值是什么？

A 3:

1. ==入栈后，LR 被更新为 EXC_RETURN（原先的 LR 被压栈，也就是有两个 LR），用在 ISR 结束后触发正常的异常返回流程==
2. 其中 [3:0] 的值可以控制 MCU 的返回模式、栈指针的选择、返回指令集状态（FreeRTOS 的任务切换正是依赖 `0xFFFFFFFD` 这个值，让每个任务拥有独立的 PSP 栈空间）
3. 被打断程序的 LR 的值是被打断函数的返回地址

### Q 4：FreeRTOS 最高能管理 NVIC 中多高的中断？如何调整 FreeRTOS 能管理的最高中断优先级的等级？

A 4:

1. 最高中断优先级的宏定义决定了 FreeRTOS 能管理的最高中断优先等级，优先级比该宏定义的值低的中断 os 都无法管理
2. 系统可管理的最高中断优先级的宏定义 configLIBRARY_MAX_SYSCALL_INTERRUPT_PRIORITY , 它是用于配置内核中的 basepri 寄存器的,当 basepri 设置为某个值的时候,NVIC 不会响应比该优先级低的中断,  而优先级比之更高的中断则不受影响

### Q 5：队列里面只能传一个 4 个字节的变量，我们一般来传要读取的变量的首地址？请问，如何要传一个结构体怎么传？

A 5:

1. 直接传指向变量的地址的指针（二级指针）
2. 将队列的字节长度修改成结构体大小（freertos 创建队列时可以指定队列 item 大小）实现内存对齐来传输结构体

###  Q 6 ：结构体指针如果要访问结构体成员应该怎么访问？

A 6:

```
typedef struct test * test 1；
//访问成员方式
test->param;
(*test).param;
```

### Q 7：extern 引用一个变量和直接用头文件包含一个变量的区别是什么？

A 7: extern 是告诉编译器这个变量在别处定义了，没在这；而在头文件中包含一个变量是这个头文件给这个变量分配了内存的（当一个项目中多次引用了同一变量时，extern 只会分配一次内存，而头文件定义会多次分配）

### Q 8：中断能否超过两个 tick 周期？为什么？

A 8: 中断不能超过两个 tick 周期，OStick 的 tick 将会丢失，导致系统时基偏移，==因为 SysTick 本身也是一个中断==。如果你的 ISR 优先级 ≥ SysTick 的优先级（注意 ARM Cortex-M 中数字越小优先级越高），那么在你的 ISR 执行期间，==SysTick 就无法抢占，tick 计数器就不会递增==，所有依赖 `vTaskDelay()`、`HAL_Delay()`、超时机制的代码，都会以为时间没过去，延时变得不准确甚至永远等待。

### Q 9：中断延迟是怎么在系统中是被什么影响的？中断延迟很高如何改善？中断延迟和中断上半部分、下半部分有什么关系？

A 9:

1. 中断延迟就是从检测到某中断请求,到执行了其服务例程的第一条指令时,已经流逝了的时间。当多个中断同时请求时,也会发生中断延迟；另外,在中断嵌套时,每个中断都会阻塞同级和低优先级的中断；最后,如果中断被掩蔽 (也就是俗称的关中,在多任务系统下满地都有),则在掩蔽期间也会附加中断延迟。
2. 上半部延迟 = 所有其他中断被延迟的时间！下半部分对中大班延迟无影响

### Q 10：S-Bus 的压栈时间受什么影响？ICode 取向量的速度优化存在哪些方法？

A 10:

1. S-Bus 的压栈时间受这些因素影响：
	1. 内存速度，数据总线访问 SRAM 的访问延迟决定了压栈速度
	2. CPU 时钟频率，频率越快指令耗时越短，压栈时间也就越短
	3. Cache 高速缓存的状态也会影响压栈时间
	4. 流水线深度，深度越深延迟越高
2. ICode 取向量的速度优化方法有：
	1. 向量表驻留 RAM
	2. 向量缓存预热，向量地址加载到 L1 Cache
	3. 只为实际使用的中断定义向量，未使用指向默认处理
