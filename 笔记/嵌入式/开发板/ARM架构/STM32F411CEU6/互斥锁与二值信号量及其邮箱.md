## 2. 逐点精讲

### 知识点 1：信号量的概念及其使用

#### 实际意义

信号主要用于任务和任务以及中断之间的同步和事件通知

#### 通俗人话解释

信号量可以理解为手机电话铃声，铃声一响就知道有电话打来

#### 关键公式/结论

1. 二进制信号量，用于互斥和同步，但是二进制信号量不会优先级继承，相比于互斥，更适用于同步，只有 0 和 1 两种状态
2. 计数信号量，多用于盘点事件如事件 a 被触发了多少次，同时用于管理系统可用资源，任务每获得信号量一次就代表使用了系统资源，信号量为 0 则没有空闲资源可用
3. 互斥量，是具有优先级继承的二进制信号量，更适用于任务之间任务与中断之间的互斥，用来保护系统资源，优先级继承机制就像是传承玉玺，谁拿到手里谁为皇帝，这时其他人的地位要低于皇帝，==注意在中断中不能使用互斥量==，其原因为互斥锁使用的优先级继承机制要求从任务中（而不是从中断中）拿、走和放入互斥锁；中断无法保持阻塞来等待一个被互斥锁保护的资源变为可用。

### 知识点 2：邮箱的概念及其作用

#### 实际意义

邮箱是用于任务之间的同步与通知，以任务通知为基础

#### 通俗人话解释

邮箱不止是通知别人有信件，它同时携带着信件，而且是有指定人的，其他人不能接收到

#### 关键公式/结论

1. 邮箱相比于队列、信号量，通知任务解除阻塞状态的速度要快 45%，消耗的 RAM 也更小
2. 同时也可以作为信号量、队列、事件组使用，不过只能 1 发送 32 位值，1. 该值保存为接收任务的 [通知值](https://freertos.org/Documentation/02-Kernel/02-Kernel-features/03-Direct-to-task-notifications/01-Task-notifications)，然后 在任何时间只能有一个通知值

### 知识点 3：环形缓冲区

#### 实际意义

用于缓存数据流，当大小不确定的缓冲区即可使用环形缓冲区

#### 应用场景

#### 常见误区

因为缓冲区成头尾相连的环形，写操作可能会覆盖未及时读取的数据，有的场景允许这种情况发生，有的场景又严格限制这种情况发生。选择何种策略和具体应用场景相关。

## 4. Q&A

### Q 1：互斥锁和二值信号量有什么区别？

 A 1:

 1. 互斥锁是一种有优先级继承机制的二值信号量，多用于互斥
 2. 二值信号量没有优先级继承机制，多用于同步

### Q 2：是否可以用二值信号量来做线程 A 和线程 B 之间的 buffer 访问的同步？

A 2: 不能，使用互斥锁的优势是优先级继承机制，这会让线程 AB 在获得互斥锁后不会被其他高优先级线程打断，导致丢数据，同时也能加快数据处理速度

### Q 3：如何线程 A 在获取互斥锁之前，不 queue_peek 去检查线程 B 的邮箱接收队列是否被邮箱取走，会发生什么情况

A 3: 线程 B 的接收队列数据还没有清空，而线程 A 发送数据给队列将旧数据覆盖，导致丢数据

### Q 4：如果线程 B 不先使用 queue_peek 去检查邮箱接收队列，直接使用 queue_receive 接收，然后去取互斥锁，将可能出现什么漏掉数据的情况？

A 4: 如果线程 B 不先使用 queue_peek 去检查邮箱接收队列，而是直接使用 queue_receive 去接收的话，可能线程 B 刚拿队列的数据，还没拿锁就被切换出去，此时线程 A 检查队列发现为空，拿锁发现可以成功获得就会误以为线程 B 已经将数据处理完毕，此时就会去启动 DMA 转换，回到线程 B 之后，很可能 buffer 的数据已经被新的 DMA 数据覆盖了，导致数据遗漏。

### Q 5：是否可以想出一种不使用互斥锁也能实现同步和数据不冲突的方案？同时也能应用到共享 Memory 来实现零拷贝？

A 5:

1. 使用无锁环形缓冲区
2. 内存池（未使用过）
